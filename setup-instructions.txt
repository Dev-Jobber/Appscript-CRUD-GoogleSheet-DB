USER MANAGEMENT CRUD APP - SETUP INSTRUCTIONS
===============================================

OVERVIEW
--------
This guide will help you set up the complete User Management CRUD application
using Google Sheets as database and Google Apps Script as backend.

ESTIMATED TIME: 15 minutes total
DIFFICULTY: Easy - Intermediate

STEP 1: CREATE GOOGLE SHEET (5 minutes)
---------------------------------------
1. Go to https://sheets.google.com
2. Click "+ Create" → "Blank spreadsheet"
3. Rename the first sheet tab to exactly: users
4. In row 1, enter these headers (A through J):
   
   A: id
   B: full_name  
   C: email
   D: phone
   E: role
   F: status
   G: created_at
   H: updated_at
   I: deleted_at
   J: is_deleted

5. Format the header row (optional):
   - Select row 1
   - Make text bold
   - Add background color (light gray)
   - Center align text

STEP 2: SETUP GOOGLE APPS SCRIPT (3 minutes)
--------------------------------------------
1. In your Google Sheet, click "Extensions" → "Apps Script"
2. A new browser tab will open with Apps Script editor
3. Delete any existing code in the editor
4. Open the file "apps-script-backend.gs" from your project folder
5. Copy the entire contents of that file
6. Paste it into the Apps Script editor
7. Click "Save" (Ctrl+S or floppy disk icon)
8. You should see "Project saved" message

STEP 3: DEPLOY AS WEB APP (2 minutes)
-------------------------------------
1. In Apps Script editor, click "Deploy" → "New deployment"
2. Click the gear icon ⚙️ next to "Select type"
3. Choose "Web app" from the dropdown
4. Fill in the deployment form:
   - Description: User Management API
   - Execute as: Me (your Google account)
   - Who has access: Anyone (recommended for testing)
5. Click "Deploy"
6. You may need to authorize permissions - click "Authorize"
7. Choose your Google account
8. Click "Advanced" → "Go to [project name] (unsafe)"
9. Click "Allow" for permissions
10. **IMPORTANT**: Copy the Web app URL that appears
    - Format: https://script.google.com/macros/s/.../exec
    - Save this URL - you'll need it in Step 4

STEP 4: CONNECT FRONTEND (1 minute)
----------------------------------
1. Open the file "app.js" from your project folder
2. Find line 4: SCRIPT_URL: 'YOUR_GOOGLE_APPS_SCRIPT_URL_HERE'
3. Replace 'YOUR_GOOGLE_APPS_SCRIPT_URL_HERE' with your actual URL from Step 3
4. Save the app.js file
5. Example of what it should look like:
   SCRIPT_URL: 'https://script.google.com/macros/s/ABC123/exec',

STEP 5: TEST THE APPLICATION (2 minutes)
----------------------------------------
1. Open "index.html" in your web browser
   - You can double-click the file or right-click → "Open with"
2. You should see the User Management System interface
3. Test creating a user:
   - Fill in all form fields
   - Click "Save User"
   - You should see success message
4. Verify data in Google Sheet:
   - Go back to your Google Sheet
   - You should see the new user data appear
5. Test other features:
   - Edit user (click Edit button)
   - Delete user (click Delete button)
   - Refresh page to see all users

TROUBLESHOOTING
---------------
Common Issues and Solutions:

1. "Sheet not found" error:
   - Make sure sheet name is exactly "users" (lowercase)
   - Run testSetup() function in Apps Script

2. "Invalid action" error:
   - Check Apps Script deployment is correct
   - Verify SCRIPT_URL is correct in app.js

3. CORS errors:
   - Make sure Apps Script is deployed as Web App
   - Check "Who has access" is set to "Anyone"

4. Data not appearing in sheet:
   - Check browser console for errors (F12)
   - Verify Apps Script execution logs
   - Make sure all form fields are filled

5. Authorization issues:
   - Make sure you clicked "Allow" for all permissions
   - Try redeploying the Web App

TESTING THE BACKEND SEPARATELY
------------------------------
You can test the Apps Script backend directly:

1. In Apps Script editor, click "Run" → "testSetup"
2. Check execution logs (Ctrl+Enter)
3. Verify test data appears in your Google Sheet
